Metodología de la página web

El objetivo de esta página web consiste en reunir todas las actividades realizadas durante el curso 2020/2021.

Requisitos necesarios:

Antes de lanzar esta página web es necesario cumplir los siguientes pasos:

  1. Pandoc: para poder realizar este ejercicio he tenido que pasar todos los archivos que tenía en formato markdown a html. Para ello, tuve que instalarme Pandoc desde la terminal mediante el comando apt cyg install, el comando que utiliza mi sistema operativo (Microsoft Windows a través de Cygwin) para instalar paquetes, el resto de sistemas operativos utilizan otros comandos como ("sudo apt install" en Linux o "brew install" en Mac OS X). Sin embargo, al pasar los archivos en formato markdown a html a través del comando pandoc los símbolos como las tildes y las eñes, por la virgulilla no eran reconocidos por el comando y, por tanto, no se traducían al formato html correctamente. Por esto mismo, tuve que cambiar los archivos de un foramto al otro a través de Pandoc Try.
  2. Bootstrap: en la página de Bootstrap me descargué los ejemplos, simplemente tuve que seleccionar en la parte superior el apartado de ejemplos y la opción de Download examples. Una vez estaban instalados utilicé el comando unzip (previamente instalado) para extraer los que a mi me interesaban que se encontraban en la carpeta assets/dist/. En la misma, había otras dos: js y css. La primera de ellas hace referencia al lenguaje java script y la segunda a la hoja de estilo que vamos a utilizar. Asimismo extraje la carpeta sticky footer navbar donde se encontraba el index.hmtl que tomé como referencia para realizar la transformación de markdown a html.
  3. Github, carpetas: cuando ya tenía todo lo que necesitaba cree una nueva carpeta en mi repositorio. Para ello, desde la terminal me situé en mi repositorio mediante el comado (cd=change directory) y con el comando (mkdir=make directory) hice la carpeta docs/. Dentro de ella mediante el comando (cp=copy) introduje las carpetas css y js. También en (docs/) copié la carpeta de imágenes que tenía subida a mi repositorio con todas las imágenes que utilice para hacer las prácticas de la asignatura. Gracias a esto pude hacer referencia a estas imágenes en mis archivos HTML ya que se encontraba dentro de docs/. Asimismo, incluí en la carpeta css la carpeta de sticky-footer-navbar.css que permitía jugar con la parte superior de la página web, donde decidí poner cada una de mis actividades. En esta carpeta también añadí cada unode mis ejercicios, ya en formato html. Despúes de realizar todas estas aciiones, en mi carpeta docs/ hay tres carpetas (css, img y js) y los ejercicios transformados en html junto a mi index.html y mi memoria.html, que corresponde justamente a este documento en el que explico la metodología.
  4. Github, pages: en el apartado Settings de mi repositorio me situé en la opción Pages. Ahí hay una ocpión que te permite establecer dónde se va a construir tu página web. Por eso, yo elegí como branchla opción main y la carpeta /docs (desde donde se construirá mi página web). Una vez elegí las opciones le di a guardar y se generó un link que me permite acceder a mi página web.

Pasos a seguir:

  1. Transformar mis archivos .md en .html Para hacer esto, como ya he explicado, tuve que utilizar la web Pandoc Try puesto que con el comando pandoc, en mi caso pandoc.exe.
  2. Copiar y pegar el contenido html en mi index.html en la parte begin content. Hay dos formas de realizar este paso. La primera de ellas, consiste en dividir el archivo index.hml en tres partes: una cabecera, un cuerpo y un pie. De esta forma, modificas la cabecera y el pie, e introduces en el cuerpo el contenido html de cada una de tus prácticas. A continuación, mediante el comando cat unes todas estas partes. Solo modificas una vez la cabecera y el pie, que será lo que utilizas todo el tiempo y el cuerpo lo vas modificando en función del contenido. Sin embargo, cuando yo realicé este ejercicio no se me ocurrió esta acción por lo que recurrí a otra táctica. Yo desde mi terminal, situada en mi repositorio y en mi carpeta docs, ponía nano (mi editor de texto) index.html, y modificaba el contenido en la parte de begin content. Realmente solo tuve que modificar una vez la cabecera de index.html donde cambié el menú superior y el pie. Así pues, son dos formas parecidas de llegar al mismo resultado.
  3. Modificar en el html algunas líneas. Sin estos cambios la página web no tendrá el diseño deseado por eso, tuve que indicarle a mi página web dónde se encontraba el estilo que yo quería usar, es decir, modifiqué las referencias que aparecen de forma automática por las que yo quería.
  4. Modificar ruta de imagen: el archivo que copiamos con formato markdown contiene imágenes que hacen referencia a una carpeta determinada porque se encuentran en nuestro repositorio. Sin embargo, cuando las copiamos en html tenemos que modificar la referencia para que muestre las imágenes que nos interesan, es decir, las que se encuentran en nuestra carpeta docs/ en otra carpeta que recibe el nombre de "img".
  5. Guardar. Cuando realicé todos estos cambios en mi archivo index.html guardé cada actividad con un nombre diferente. De esta forma, se quedaban mi archivo index.html y los nuevos que iba creando.

¿Cómo acceder a la página web?

    Existen varias formas de acceder a la página web resultante de la carpeta docs/:

  • Atajo de teclado: control+o, que se utiliza para abrir una URL, un documento, una imagen desde el ordenador. Para ello, debes situarte en el navegador de internet, presionar las teclas mencionadas y elegir aquello que quieres abrir, en mi caso un archivo html.
  • Github: te sitúas en Settings luego seleccionas Pages y te aparece el link en la parte superior, mediante el cual puedes acceder directamente.

Paso final

Al finalizar todo este proceso comprimí mi repositorio, "uc3m-periodismo-datos" en una carpeta zip.Esto lo hice a través del comando zip desde fuera de la carpeta puse "zip -r cursocompleto.zip docs/." Esto quiere decir que quería que mi carpeta se llamara "Curso Completo" y la carpeta que quería zipear era docs. A continuación, copie ese .zip en mi carpeta docs, lo añadí a mi repositorio git y lo subí a Github mediante los comandos git add, git commit -m, git push origin main.
